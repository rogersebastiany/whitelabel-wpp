WHITELABEL-WPP — POC Report
Date: 2026-04-16
Status: Local POC running, pipeline validated end-to-end

========================================
PROJECT SUMMARY
========================================

WhatsApp Group Analytics Platform — white-label SaaS that listens to group
conversations, extracts insights (topics, engagement, key members), and lets
group owners query their data through a 1:1 WhatsApp chat.

LGPD compliant: no raw messages stored. Only metadata, topics, and aggregated
summaries are persisted.

1. Initial scaffold — README, pyproject.toml, .env.example
2. CloudFormation stack — API Gateway, Lambda, SQS, EventBridge, CloudWatch
3. Full infra deployed — Secrets Manager, ECS Fargate (Spot, auto-scale 0→3),

5. LanceDB role fix — sync bridge between Cognee and Milvus
6. Multi-tenancy spec — shared instances, partition by group_id
7. Telnyx eSIM onboarding spec — programmatic provisioning
8. Switch to Telnyx BSP — single vendor (numbers, WABA, messaging, webhooks)
9. Core processor — config, models, all clients, processor dispatch, owner
   chat handler, webhook handler
10. Local POC — Go whatsmeow bridge, docker-compose (Neo4j + Milvus + etcd +
    MinIO + Attu), dual-mode config, WhatsmeowClient
11. Intent tuning — silence unknown intents, add PT-BR keywords
12. POC hardening — sent-ID echo filter, OpenAI topic extraction, data-driven
    summaries with short LLM blurb

========================================
WHAT'S WORKING
========================================

✓ WhatsApp group messages received via whatsmeow bridge
✓ Interaction metadata stored in Neo4j (no raw text — LGPD)
✓ Topic extraction via OpenAI GPT-4o-mini
✓ Topics stored in Neo4j + Milvus with semantic dedup (cosine > 0.85)
✓ Owner 1:1 chat interface (summary, topics, key members, engagement)
✓ Reply loop prevention (sent-ID tracking)
✓ Multi-member detection from group messages
✓ Reply-to context tracking
